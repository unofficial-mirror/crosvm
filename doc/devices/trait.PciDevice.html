fn assign_irq(
    &mut self,
    _irq_evt: &Event,
    _irq_resample_evt: &Event,
    _irq_num: Option<u32>
) -> Option<(u32, PciInterruptPin)>[src]

Assign a legacy PCI IRQ to this device. The device may write to irq_evt to trigger an interrupt. When irq_resample_evt is signaled, the device should re-assert irq_evt if necessary. Optional irq_num can be used for default INTx allocation, device can overwrite it. If legacy INTx is used, function shall return requested IRQ number and PCI INTx pin.

fn allocate_io_bars(
    &mut self,
    _resources: &mut SystemAllocator
) -> Result<Vec<(u64, u64)>, Error>[src]

Allocates the needed IO BAR space using the allocate function which takes a size and returns an address. Returns a Vec of (address, length) tuples.

fn allocate_device_bars(
    &mut self,
    _resources: &mut SystemAllocator
) -> Result<Vec<(u64, u64)>, Error>[src]

Allocates the needed device BAR space. Returns a Vec of (address, length) tuples. Unlike MMIO BARs (see allocate_io_bars), device BARs are not expected to incur VM exits

  • these BARs represent normal memory.
