Nested Target Extension - Set/Remove Software Breakpoints.

See this stackoverflow discussion about the differences between hardware and software breakpoints.

Recommendation: If you're implementing Target for an emulator that's using an interpreted CPU (as opposed to a JIT), the simplest way to implement "software" breakpoints would be to check the PC value after each CPU cycle, ignoring the specified breakpoint kind entirely.
